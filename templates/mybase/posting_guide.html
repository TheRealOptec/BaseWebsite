How to Make Posts

This page explains the posting flow for the forum and the exact Simples/XML syntax currently supported by the editor and compiler.

Recommended Workflow

1

Pick the right topic

Make sure your post belongs in the correct discussion area before you start writing.

2

Write the raw post content

Enter your content in the editor using the raw Simples/XML syntax when needed.

3

Preview the compiled output

Use the preview mode to check headings, paragraphs, and embedded content before submitting.

4

Submit when it looks right

If the preview looks correct and the post stays on topic, you can submit it.

Syntax Rules

A

Use real Simples tags

The compiler currently supports <h> and <p> for text structure. If you use other tag names, they may not render.

B

Wrap structured content in <simples>

A structured post should use a single <simples> root element, with your headings, paragraphs, and embeds nested inside it.

C

Keep the XML valid

Open and close each tag properly, and make sure nested elements are placed inside the correct parent tag.

D

Use <embed> only where text content can appear

The embed container does not display text by itself. It should be placed in a normal content flow, not nested inside another <embed>.

Basic Simples Example

A post can still be plain text, but if you want structured output the raw source should look more like this: 

<simples>
  <h>My first post</h>
  <p>This is the main body of the post.</p>
  <p>You can add more than one paragraph if needed.</p>
</simples>

News Embed Example

The current documentation and compiler support a <news> embed inside <embed>. The request details are provided through the child tags below. 

<simples>
  <h>Latest AI News</h>
  <p>Here are some matching headlines:</p>
  <embed>
    <news>
      <q>artificial intelligence</q>
      <from>2026-03-10</from>
      <language>en</language>
      <sortBy>publishedAt</sortBy>
      <top>3</top>
    </news>
  </embed>
</simples>
Tag What it does
<q> Sets the news search query.
<from> Sets the earliest date for returned articles. Use yyyy-mm-dd.
<language> Sets the article language, commonly en for English.
<sortBy> Sets the sorting option passed to the news request.
<top> Limits how many articles are displayed.

Good Posting Tips

  • Use a clear title so people know what your post is about.
  • Keep your post respectful and useful to the discussion.
  • Only add embedded content when it genuinely supports the post.
  • Preview first if you are using Simples/XML formatting.
  • Double-check your closing tags if the preview does not look right.
  • Keep API examples concise so topic pages stay readable.
  • Avoid duplicate posts if a similar thread already exists.
