How to Make Posts

This page explains the expected posting flow for the forum and how the Simples/XML editor is meant to be used.

Recommended Workflow

1

Pick the right topic

Make sure your post belongs in the correct discussion area before you start writing.

2

Write the raw post content

Enter your content in the editor using the raw Simples/XML syntax when needed.

3

Preview the compiled output

Use the preview mode to check headings, paragraphs, and embedded content before submitting.

4

Submit when it looks right

If the preview looks correct and the post stays on topic, you can submit it.

Simple Example

A post can still be plain text, but if you want structured output the raw source might look something like this: 

<simples>
  <header>My first post</header>
  <paragraph>This is the main body of the post.</paragraph>
</simples>

Good Posting Tips

  • Use a clear title so people know what your post is about.
  • Keep your post respectful and useful to the discussion.
  • Only add images or embedded content when they support the post.
  • Preview first if you are using Simples/XML formatting.
  • Avoid duplicate posts if a similar thread already exists.
